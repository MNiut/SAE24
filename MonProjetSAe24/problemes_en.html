The Challenges of the SAÃ©24 Project

Every technical project has its share of challenges. This page documents the main obstacles we encountered and the solutions we implemented to overcome them.

DHCP Server Conflict

Problem: At the beginning of the project, our client machines were not getting IP addresses from our Windows Server, but were receiving them from the lab's network.

Solution: By isolating our topology on a dedicated vSwitch in ESXi and using client machines physically connected only to our own switch, we created an isolated network environment where our DHCP server was the sole master.

SIP Trunk Configuration for Intersite Telephony

Problem: Phone calls between our Asterisk servers would not go through. Analyzing the logs (asterisk -rvvv) revealed an authentication error.

Solution: The issue stemmed from an inconsistency in the "secrets" (passwords) and "contexts" defined in each server's sip.conf file. After rigorous coordination with the other team, the calls could be established.

Time Management and Task Dependencies

Problem: We had underestimated the time required for configuring the Linux services, which delayed the start of the website development.

Solution: We readjusted our Gantt chart. The person in charge of the website started HTML/CSS development locally (using XAMPP) without waiting for the server to be 100% ready. This parallelization of tasks allowed us to catch up.
